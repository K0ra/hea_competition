# Model runs

Model outputs and run records live here. Each run is documented as a **JSON file** with:

- **task**: target, metrics
- **data**: source, split (train/val waves), features, preprocessing
- **model**: type and hyperparameters
- **results**: validation metrics (F2, PR-AUC, ROC-AUC, best threshold)

## Files

- **`baseline_run.json`** – Diabetes baseline (backward compat). Same content as `baseline_run_DIABE.json`.
- **`baseline_run_<suffix>.json`** – Per-condition baseline (DIABE, HIBPE, CANCR, LUNGE, HEARTE, STROKE, PSYCHE, ARTHRE). Generated by `scripts/step06_train_baseline.py`.

## Interpreting results

See [../docs/metrics_guide.md](../docs/metrics_guide.md) for F2, PR-AUC, and ROC-AUC.

## Adding new runs

Re-run `step06_train_baseline.py` to overwrite `baseline_run.json`, or write new JSON files (e.g. `run_002.json`) with different `run_id` and model/split to compare.
